🔴 The 117% bug — found the exact cause
It's not a display bug, it's a real logic bug in phase4-agent/analyzer.py:
sentiment_summary percentages are computed from all rows ever saved to analysis_results (cumulative across every run you've triggered).
But total_analyzed saved to pipeline_runs is set to total_to_analyze — the count of only newly analyzed reviews in that specific run (line ~376).
Your frontend (app.js line ~178) then does (pos+mix)/total_analyzed * 100 — dividing a cumulative numerator by an incremental-only denominator. If a run only analyzed a few new reviews but the cumulative sentiment counts are much larger, you get >100%.
Fix: in update_pipeline_run(...), change total_analyzed=total_to_analyze to total_analyzed=len(all_classification) (the cumulative count already computed a few lines earlier). One-line fix, but it's the reason your headline KPI is broken.
